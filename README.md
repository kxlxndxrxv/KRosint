# 🔍 KRosint

### Анализ эволюции никнеймов IT-сообществ (2000–2026)

**Программный комплекс для OSINT-исследования юзернеймов**
Собирает 5000 датированных никнеймов из 4 источников и выявляет, как менялась структура имён пользователей за 26 лет.

## 💡 О проекте

Юзернейм — самый устойчивый элемент цифрового следа. Пользователь меняет фотографию, биографию, email, но имя учётной записи остаётся неизменным десятилетиями. И сама строка никнейма несёт массу скрытой информации — в том числе **о том, когда был создан аккаунт**.

KRosint автоматически собирает никнеймы из четырёх IT-сообществ за 26 лет, разбивает их по пяти эпохам и выявляет закономерности — как никнейм `wizard` из 2003-го превратился в `dark_coder_99` к 2024-му.

## 🚀 Быстрый старт

### 🪟 Для Windows (без установки Python)

Скачайте **[KRosint v1.0](https://github.com/kxlxndxrxv/KRosint/releases/tag/v1.0)** из раздела Releases, распакуйте архив и запустите `KRosint.exe` двойным щелчком. Приложение откроется в браузере автоматически — установка Python и зависимостей не требуется.

### 🛠️ Для разработчиков

```bash
git clone https://github.com/kxlxndxrxv/KRosint.git
cd KRosint
pip install -r requirements.txt
streamlit run main.py
```

Откроется вкладка в браузере — нажмите **«📡 Собрать данные»**, дождитесь завершения, затем **«📊 Провести анализ»**.

## ✨ Возможности

- 🌐 **Многоисточниковый сбор** — параллельный обход 4 платформ через Wayback Machine CDX API и публичные REST/Firebase API
- 🎯 **Сбалансированный датасет** — ровно 1000 никнеймов на каждый из 5 периодов (всего 5000)
- 📐 **14 структурных признаков** — длина, доля цифр, разделители, регистр, лит-спик и др.
- 🎨 **5 структурных масок** — словарный, слово+цифры, смешанный, спецсимвол в начале, CamelCase
- 🤖 **ML-классификатор** — Random Forest для определения периода создания никнейма
- 📊 **7 типов визуализаций** — графики, тепловые карты, диаграммы размаха
- 💾 **CDX-кэш** — повторные запуски не дёргают веб-архив без необходимости
- ⚡ **Параллельная обработка** — ThreadPoolExecutor ускоряет HackerNews API в 6–8 раз
- 🖥️ **Веб-интерфейс на Streamlit** — без CLI и конфигов

## 📡 Источники данных

Все четыре источника принадлежат IT-нише, что обеспечивает методологическую чистоту сравнения периодов:

| Источник | Тематика | Год основания | Метод сбора | Покрытие |
|----------|----------|---------------|-------------|----------|
| **Slashdot** | IT-новости, форум | 1997 | Wayback Machine CDX API | 2000–2026 |
| **SourceForge** | Open-source | 1999 | Wayback Machine CDX API | 2000–2026 |
| **HackerNews** | IT, стартапы | 2007 | Firebase API | 2007–2026 |
| **GitHub** | Разработка ПО | 2008 | REST API | 2008–2026 |

### Периодизация (IT-ориентированная)

| Период | Эпоха в IT |
|--------|-----------|
| **2000–2004** | Ранний Web 1.0, культура UNIX-логинов, HTML-форумы |
| **2005–2009** | Web 2.0, запуск Stack Overflow, GitHub и HackerNews |
| **2010–2014** | GitHub становится стандартом open-source, никнейм = бренд разработчика |
| **2015–2019** | Массовая аудитория на GitHub, исчерпание коротких имён |
| **2020–2026** | Насыщение пространства имён, эра составных конструкций |

## 📊 Ключевые находки

На датасете 5000 никнеймов за 2000–2026 гг.:

| Метрика | 2000–2004 | 2020–2026 | Изменение |
|---------|-----------|-----------|-----------|
| Средняя длина | 7,4 симв | 9,2 симв | +24% |
| Содержат цифры | 14% | 44% | **×3,1** |
| Лит-спик | 12% | 38% | **×3,2** |
| Полностью lowercase | 81% | 67% | −14 п.п. |
| Доминирующая маска | словарный (59,7%) | паритет word/mixed | смена парадигмы |

**Открытие:** «словарная» маска (вид `wizard`) выросла до пика 71,5% в 2005–2009 гг., затем обрушилась до 31% в 2020–2026 — переломный момент пришёлся на 2015 год, когда GitHub вышел за пределы узкого open-source.

ML-классификатор достигает точности **30–40%** при пяти классах (базис 20%) и **~75%** для бинарной задачи «до 2015 / после 2015».

## 🖥️ Интерфейс

Двухколоночный Streamlit-интерфейс:

- **Слева** — две кнопки управления и текстовый отчёт

## 📖 Методика

Шестиэтапная методика структуризации:

1. **Идентификация** — выбор IT-платформ и временны́х периодов
2. **Сбор** — параллельный обход 4 API с кэшированием
3. **Балансировка** — ровно 1000 никнеймов на период
4. **Признаки** — 14 структурных метрик + 5 композиционных масок
5. **Анализ** — статистика и визуализация
6. **ML** — Random Forest (200 деревьев), 5-fold кросс-валидация

Полный текст методики и обоснование выбора признаков — в [курсовой работе](docs/coursework.pdf).

## ⚠️ Ограничения

- Все источники — IT-сообщества, поэтому выводы не переносятся напрямую на форумы других тематик
- Датировка через Wayback Machine **косвенная**: ник существовал не позже даты снапшота
- Для GitHub и HackerNews API есть rate limit — первый сбор может занять 15–30 минут
- На повторных запусках CDX-кэш экономит трафик и время
